# On reading plain text

Every format eventually asks you to trust its editor. Markdown asked for
less: a text file, some punctuation for structure, and the promise that any
machine on earth could open it in fifty years. That promise is the whole
appeal. It is also why almost nothing on the Mac shows it properly.

## The wrong app, by default

Double-click a `.md` file and you get raw source in TextEdit, or a code
editor treating your reading material like a file to be modified. Both miss
the point. A manuscript is not a draft you happen to be viewing; once it is
finished, or once someone else wrote it, the only honest interaction left is
reading.

> Printers have known this for centuries. Before a book exists, its
> manuscript is set in type and printed once, plainly, so people can read
> it. That copy is called a galley.

## What changes when nothing can be edited

Read-only is not a missing feature. It is the design. There is no cursor to
lose, no autosave to distrust, no accidental keystroke that turns a shared
document into a diff. When a typo does need fixing, edit mode is one
keystroke away, and leaving it is just as fast, because writing and reading
are different tasks and deserve different modes.

## Living documents

The files worth reading today are rarely finished. An agent is still
writing the plan. A log is still growing. Galley watches the file, redraws
the page on every save, and keeps your place while it happens[^1]. A small
pill says updated. That is the whole ceremony[^2].

## The terminal, typeset

This theme sets everything in mono, on purpose. For people who already live
in one, a monospace reading surface is not a compromise. It is home.

[^1]: Scroll position is preserved unless you were already at the bottom, in
    which case the page follows the tail like a log.
[^2]: No progress bar, no spinner, no toast stack. One pill, one word.
